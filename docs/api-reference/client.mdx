# SDK Client

The TypeWoo SDK is exported as a singleton instance. This page documents the main client interface.

## Import

```typescript
import { Typewoo } from '@typewoo/sdk';
```

## Initialization

### `init(config: SdkConfig): Promise<void>`

Initialize the SDK with configuration options. Must be called before accessing any services.

```typescript
await Typewoo.init({
  baseUrl: 'https://your-store.com',
});
```

**Parameters:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `config` | `SdkConfig` | Yes | SDK configuration object |

**SdkConfig Interface:**

```typescript
interface SdkConfig {
  baseUrl: string;
  admin?: {
    consumer_key?: string;
    consumer_secret?: string;
    useAuthInterceptor?: boolean;
  };
  auth?: {
    autoLoginUrl?: string;
    fetchCartOnLogin?: boolean;
    revokeTokenBeforeLogin?: boolean;
    accessToken?: {
      key?: string;
      disabled?: boolean;
      storage?: StorageType | StorageProvider;
      useInterceptor?: boolean;
    };
    refreshToken?: {
      key?: string;
      disabled?: boolean;
      storage?: StorageType | StorageProvider;
      useInterceptor?: boolean;
    };
  };
  nonce?: {
    key?: string;
    disabled?: boolean;
    storage?: StorageType | StorageProvider;
  };
  cartToken?: {
    key?: string;
    disabled?: boolean;
    storage?: StorageType | StorageProvider;
  };
  axiosConfig?: AxiosRequestConfig;
  request?: {
    retry?: RetryConfig;
  };
}
```

**Returns:** `Promise<void>`

**Notes:**

- Initialization is idempotent; subsequent calls are ignored
- Must complete before accessing services, state, or config

## Properties

### `store`

Access Store API services (WooCommerce Store API endpoints).

```typescript
const products = await Typewoo.store.products.list();
const cart = await Typewoo.store.cart.get();
```

**Available Services:**

| Property | Service | Description |
|----------|---------|-------------|
| `products` | `ProductService` | Product listing and details |
| `cart` | `CartService` | Cart operations |
| `cartItems` | `CartItemService` | Cart item CRUD |
| `cartCoupons` | `CartCouponService` | Coupon management |
| `cartExtensions` | `CartExtensionsService` | Cart extensions |

### `auth`

Access Authentication API services.

```typescript
await Typewoo.auth.token({ login: 'user', password: 'pass' });
await Typewoo.auth.validate();
```

**Available Methods:**

| Method | Description |
|--------|-------------|
| `token(body)` | Obtain access and refresh tokens |
| `refreshToken(body)` | Refresh an access token |
| `revokeToken(body?)` | Revoke/logout |
| `validate()` | Validate current token |
| `status()` | Get authentication status |
| `oneTimeToken(body)` | Generate one-time login token |
| `getAutoLoginUrl(ott, redirect, params?)` | Build auto-login URL |

### `events`

Access the EventBus for subscribing to SDK events.

```typescript
Typewoo.events.on('cart:updated', (cart) => {
  console.log('Cart updated:', cart);
});
```

**Type:** `EventBus<SdkEvent>`

See [Hooks and Events](/hooks-and-events) for full documentation.

### `state`

Access the SDK's runtime state.

```typescript
console.log('Authenticated:', Typewoo.state.authenticated);
console.log('Cart token:', Typewoo.state.cartToken);
```

**Type:** `SdkState`

```typescript
interface SdkState {
  cart?: CartResponse;
  nonce?: string;
  cartHash?: string;
  cartToken?: string;
  authenticated?: boolean;
}
```

### `config`

Access the resolved SDK configuration.

```typescript
console.log('Base URL:', Typewoo.config.baseUrl);
```

**Type:** `ResolvedSdkConfig`

**Note:** Throws an error if accessed before initialization.

## HTTP Client

The SDK exports HTTP client utilities for advanced use cases:

```typescript
import { httpClient, createHttpClient } from '@typewoo/sdk';
```

### `httpClient`

A proxy to the underlying Axios instance. Throws if accessed before SDK initialization.

### `createHttpClient(config)`

Creates the Axios instance. Called internally during `init()`. Idempotent.

## Error Handling

All SDK methods return an `ApiResult` object with `data` or `error`:

```typescript
const { data, error } = await Typewoo.store.products.list();

if (error) {
  console.error('Error code:', error.code);
  console.error('Message:', error.message);
  console.error('HTTP status:', error.data.status);
  return;
}

// data is defined here
console.log('Products:', data);
```

See [Error Handling](/error-handling) for detailed information.

## Complete Example

```typescript
import { Typewoo, ProductResponse, CartResponse } from '@typewoo/sdk';

// Initialize
await Typewoo.init({
  baseUrl: 'https://your-store.com',
  auth: {
    accessToken: {
      storage: 'localstorage',
    },
  },
});

// Subscribe to events
Typewoo.events.on('auth:changed', (isAuth) => {
  console.log('Auth changed:', isAuth);
});

Typewoo.events.on('cart:updated', (cart) => {
  updateCartUI(cart);
});

// Authenticate
const { error: authError } = await Typewoo.auth.token({
  login: 'user@example.com',
  password: 'password',
});

if (authError) {
  console.error('Login failed');
} else {
  console.log('Logged in, authenticated:', Typewoo.state.authenticated);
}

// Use Store API
const { data: products } = await Typewoo.store.products.list({
  per_page: 10,
  orderby: 'popularity',
});

const { data: cart } = await Typewoo.store.cart.add({
  id: products![0].id,
  quantity: 1,
});

console.log('Cart items:', cart?.items_count);
```
